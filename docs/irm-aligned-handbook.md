# IRM-Aligned Handbook Scaffold

## Purpose

This handbook organizes repository guidance in an IRM-inspired structure for engineering, operations, and governance teams.

## Sections

1. **Program Administration**: ownership, approvals, and change management.
2. **Identity and Authorization**: client IDs, secret handling, operator access, and evidence requirements.
3. **Transmission Operations**: gateway controls, tunnel prerequisites, acknowledgements, and exception handling.
4. **Transcript and TDS Operations**: transcript pull workflow, TDS cadence, and masterfile processing checkpoints.
5. **Refund Status Operations**: event-driven updates, approved source management, and non-scraping controls.
6. **Analytics and Intelligence**: TC code indicators, analytics center outputs, and auditability.
7. **Forms and Letters Governance**: template stewardship, approvals, and release controls.

## Operating Principle

Every future implementation must link technical changes to an approved operating procedure, security review, and compliance checkpoint.
